# Prompt Padrão de Desenvolvimento
Projeto: Escola de Pais Online

## Documentação Oficial

Considere os documentos localizados na pasta `docs` como a única fonte oficial do projeto.

Antes de iniciar qualquer implementação, leia completamente:

- docs/Documentacao_Projeto_Escola_de_Pais_Online.md
- docs/Checklist_Projeto_Blog_Escola_de_Pais_Online.md
- docs/Manual_de_Design_e_Interface_Escola_de_Pais_Online.md
- docs/CHANGELOG.md

Nenhuma implementação poderá contrariar estes documentos.

---

## Regras Gerais

Sempre:

- respeitar rigorosamente a arquitetura definida;
- seguir o Manual de Design e Interface;
- seguir o Checklist de Implementação;
- nunca alterar decisões arquiteturais já aprovadas;
- implementar apenas a Feature solicitada;
- utilizar HTML, CSS e JavaScript puro;
- manter compatibilidade com Cloudflare Pages;
- manter SEO, acessibilidade e responsividade;
- manter componentes reutilizáveis;
- evitar duplicação de código;
- preservar a organização do projeto.

---

## Atualização da Documentação

Após concluir uma Feature e somente após sua aprovação:

Atualizar automaticamente:

- docs/CHANGELOG.md
- docs/Checklist_Projeto_Blog_Escola_de_Pais_Online.md

Nunca alterar automaticamente:

- docs/Documentacao_Projeto_Escola_de_Pais_Online.md
- docs/Manual_de_Design_e_Interface_Escola_de_Pais_Online.md

Esses documentos somente poderão ser modificados mediante autorização explícita.

---

## Ao finalizar uma Feature

Sempre informar:

### Arquivos criados

### Arquivos alterados

### Arquivos removidos (se houver)

### Documentos atualizados

### Testes realizados

### Observações importantes

Nunca avançar automaticamente para a próxima Feature.

## Restrições

Nunca:

- alterar Features já aprovadas sem autorização;
- modificar documentação arquitetural automaticamente;
- fazer commit;
- executar push;
- alterar arquivos não relacionados à Feature atual;
- adicionar dependências externas sem autorização.

Sempre trabalhar apenas na Feature informada.
